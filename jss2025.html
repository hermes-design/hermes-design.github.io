Example PRISM-games CSG Code

Here's a simple example of a CSG (Concurrent Stochastic Game) model in PRISM-games syntax:


// example_csg.prism
// A simple game with two players

csg // Model type declaration

player p1 // Player 1
  module M1
    x : [0..1] init 0; // Player 1's variable
    [a] x=0 -> (x'=1); // Action 'a' available to p1 in state x=0
    [b] x=1 -> (x'=0); // Action 'b' available to p1 in state x=1
  endmodule
endplayer

player p2 // Player 2
  module M2
    y : [0..1] init 0; // Player 2's variable
    [c] y=0 -> (y'=1); // Action 'c' available to p2 in state y=0
    [d] y=1 -> (y'=0); // Action 'd' available to p2 in state y=1
  endmodule
endplayer

// Global variables and transitions (optional, can be used for environment/shared state)
module Environment
  z : [0..1] init 0;
  // Example of a probabilistic transition influenced by player actions
  // This is a simplified representation; actual CSG transitions can be more complex
  [a,c] z=0 -> 0.5:(z'=1) + 0.5:(z'=0); // If p1 plays 'a' and p2 plays 'c'
  [b,d] z=1 -> (z'=0);
endmodule

// Rewards for player 1 (p1)
rewards "goal_for_p1" player p1
  x=1 & y=0 & z=1 : 1; // p1 gets a reward of 1 if this state is reached
endrewards

// Rewards for player 2 (p2) - can be defined similarly
// rewards "goal_for_p2" player p2
//   x=0 & y=1 : 1; 
// endrewards

To analyze this CSG model, you would save it (e.g., as example_csg.prism) and then use PRISM-games to load it. Properties are typically specified using rPATL (reward Probabilistic Alternating-time Temporal Logic). For example, an rPATL property could be:


// example_properties.pctl
// Can player p1 ensure that eventually x=1 and y=0 and z=1 with probability 1?
// (Assuming "goal_for_p1" is defined to give reward 1 in that state and 0 otherwise)
<<p1>> Pmax=? [ F "goal_for_p1" ]

// What is the maximum probability that player p1 can ensure eventually reaching a state
// where x=1, regardless of player p2's actions?
<<p1>> Pmax=? [ F (x=1) ]

// Minimum expected reward for player p1 to reach "goal_for_p1"
// (This might require specific reward structures and interpretations in CSGs)
// <<p1>> Rmin{"goal_for_p1"}=? [ F (x=1 & y=0 & z=1) ]

Note: CSG modeling and rPATL property specification can be quite complex. The example above is a basic illustration. Refer to the PRISM-games manual for detailed syntax and capabilities.

UCP (Use Case Points) Analysis Guide

About ucp_reader.py

The ucp_reader.py script is a Python utility designed to calculate Use Case Points (UCP) based on input data provided in a JSON file. It computes Unadjusted Use Case Weight (UUCW), Unadjusted Actor Weight (UAW), Technical Complexity Factor (TCF), Environmental Complexity Factor (ECF), and the final Use Case Points (UCP). Additionally, it generates an estimated effort in hours based on a range of "hours per UCP" values and saves this data to a CSV file.

How to Use ucp_reader.py

To run the ucp_reader.py script, you need to provide the path to your UCP data JSON file as a command-line argument. The script will then process this file, perform the calculations, print a summary to the console, and generate a CSV file named ucp_estimated_effort.csv in the same directory.

Command Line Usage:


python3 ucp_reader.py ucp_data.json

Replace ucp_data.json with the actual path to your JSON input file.
