Modeling CTMC in PRISM: Enqueue and Dequeue System

This example models a simple queueing system using a Continuous-Time Markov Chain (CTMC). Items arrive at a queue with a certain rate (enqueue), and are processed and leave the queue with another rate (dequeue).

Example CTMC Model: Enqueue/Dequeue Queue


// enqueue_dequeue_ctmc.prism
// Defines a Continuous-Time Markov Chain (CTMC) model

ctmc

// Constant for the maximum capacity of the queue
const int MAX_CAPACITY = 5;

// Module defining the queue behavior
module Queue
  // State variable 'count' represents the number of items in the queue
  count : [0..MAX_CAPACITY] init 0; // Initial count is 0

  // Transition for enqueueing an item
  // If the queue is not full, an item can be enqueued
  [enqueue] count < MAX_CAPACITY -> rate_enqueue : (count' = count + 1);

  // Transition for dequeueing an item
  // If there are items in the queue, an item can be dequeued
  [dequeue] count > 0 -> rate_dequeue : (count' = count - 1);
endmodule

// Rates for enqueue and dequeue operations
const double rate_enqueue = 0.2; // Rate at which items arrive
const double rate_dequeue = 0.3; // Rate at which items are processed

// Rewards structure to track the queue length
rewards "queue_length"
  true : count; // The reward is the current number of items in the queue
endrewards

Example CTMC Properties for Enqueue/Dequeue System


// enqueue_dequeue_properties.csl
// Properties to verify on the CTMC model

// P=? [ S=50 ] (count=MAX_CAPACITY)
// Probability that the queue reaches its maximum capacity (MAX_CAPACITY) within 50 time units.
P=? [ S=50 ] (count=MAX_CAPACITY)

// R{"queue_length"}=? [ S=100 ]
// Expected cumulative reward (total queue length over time) within 100 time units.
R{"queue_length"}=? [ S=100 ]

// R{"time"}=? [ F (count=MAX_CAPACITY) ]
// Expected time until the queue first reaches its maximum capacity.
R{"time"}=? [ F (count=MAX_CAPACITY) ]

// S=? [ count=0 ]
// Steady-state probability that the queue is empty.
S=? [ count=0 ]

// R{"queue_length"}=? [ S ]
// Expected steady-state value of the queue length.
